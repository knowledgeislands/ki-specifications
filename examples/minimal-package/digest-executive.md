# Executive digest

This package scans how other standards bodies number their normative documents, so that the numbering scheme adopted here (`KIP-NNNNNN` for proposals, `KIS-NNNN` for specifications) can be chosen with the trade-offs made explicit. It compares sequential integer numbering (RFC-style), date-prefixed numbering (as used by some W3C notes), and category-prefixed numbering (as used by IETF area-specific series), against the needs of a small, slow-moving specification repository with a public proposal-then-specification pipeline. The short answer: sequential integer numbering, split into a wide proposal series and a narrow specification series, gives the clearest signal of maturity with the least ongoing administrative overhead.
