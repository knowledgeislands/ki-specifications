# Examples

Two fully worked Knowledge Packages, each schema-valid and each satisfying the checklist for the [conformance level](../specifications/KIS-0001-knowledge-package/conformance.md) it declares.

| Package | Conformance level | Subject |
| --- | --- | --- |
| [`minimal-package/`](minimal-package/) | Minimal | Evaluating specification numbering schemes for a standards repository |
| [`full-package/`](full-package/) | Extended | The Knowledge Islands AI Operating System: kernel, kernel modules, federated model execution, and hierarchical continual improvement |

`minimal-package/` shows the smallest useful package: the required core fields and a single executive digest.

`full-package/` shows a mature package that has been through decisions, tasks, review, and a handover, with every Extended-level field populated and the technical digest, decisions, and handover material carried as real Markdown artefacts alongside `manifest.json`.

Compare either against [`../schemas/examples/`](../schemas/examples/) for the terse schema-conformance fixtures, or against [`../templates/`](../templates/) for a blank starting point.
